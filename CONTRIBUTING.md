# 🤝 Contributing to Open Source Project Manager CLI

Thank you for your interest in contributing! 🎉
This project is beginner-friendly and welcomes all types of contributions.

---

## 📌 Ways You Can Contribute

You can help improve this project in many ways:

* 🐛 Fix bugs
* ✨ Add new features
* 📝 Improve documentation
* 🎨 Refactor or clean code
* 📊 Add better data handling or validation
* 💡 Suggest ideas via Issues

---

## 🚀 Getting Started

### 1. Fork the Repository

Click the **Fork** button on the top right of this repository.

---

### 2. Clone Your Fork

```bash
git clone https://github.com/your-username/project-name.git
cd project-name
```

---

### 3. Create a Branch

Always create a new branch for your changes:

```bash
git checkout -b feature/your-feature-name
```

Example:

```bash
git checkout -b fix/input-validation
```

---

## 🛠️ Development Guidelines

* Follow clean and readable Python code practices
* Use meaningful variable names
* Add comments where necessary
* Keep functions small and simple
* Avoid breaking existing features

---

## 📂 Project Structure (Important)

* `task_5.py` → Main program file
* Generated folders → Output files (auto-created)

---

## ✅ Before Submitting

Make sure:

* Your code runs without errors
* You tested your changes
* You didn’t break existing functionality
* Your commit message is clear

---

## 💬 Commit Message Format

Use simple and clear messages:

```
Added feature: contributor filtering
Fixed bug: issue count not updating
Improved: README documentation
```

---

## 📤 Submitting Your Contribution

### Step 1: Add and Commit

```bash
git add .
git commit -m "Your message here"
```

---

### Step 2: Push Changes

```bash
git push origin feature/your-feature-name
```

---

### Step 3: Create Pull Request (PR)

* Go to your fork on GitHub
* Click **"Compare & Pull Request"**
* Describe your changes clearly
* Submit the PR

---

## 🔍 Code Review Process

* Maintainers will review your PR
* You may be asked to make changes
* Once approved, your contribution will be merged 🎉

---

## ⚠️ Rules

* ❌ Do not commit directly to `main`
* ❌ Do not submit unrelated changes in one PR
* ❌ Do not copy code without understanding

---

## 💡 Suggested Contributions (Good First Issues)

* Add input validation
* Improve CLI UI (better formatting)
* Add search/filter for issues
* Export data in JSON format
* Add logging system

---

## 🙌 Thank You

Your contributions help make this project better and help you grow as a developer 🚀

Happy coding!
